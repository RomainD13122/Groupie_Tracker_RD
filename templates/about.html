À propos de PokéTracker

Présentation du projet

PokéTracker est une application web permettant d'explorer et de suivre les cartes Pokémon. Cette application a été développée dans le cadre du projet "Groupie Tracker" pour démontrer l'utilisation d'une API REST et l'implémentation de fonctionnalités web essentielles en Go.

Grâce à PokéTracker, les utilisateurs peuvent explorer des milliers de cartes Pokémon, rechercher, filtrer et ajouter leurs cartes préférées à leur liste de favoris. L'application utilise l'API TCGdex pour récupérer les informations sur les cartes et les collections.

Fonctionnalités principales

Recherche de cartes par nom
Filtrage par type, rareté et collection
Pagination des résultats
Gestion des cartes favorites
Exploration des collections
Détails des cartes

FAQ sur la gestion du projet

Comment avez-vous décomposé le projet ? Quelles ont été les phases clé ?

Le projet a été structuré en plusieurs phases distinctes pour assurer une progression méthodique :

Phase 1 : Recherche et planification (3 jours)

J'ai commencé par explorer différentes API liées aux cartes Pokémon et j'ai finalement choisi l'API TCGdex pour sa richesse en données et sa documentation. J'ai analysé sa structure, ses endpoints et les formats de données qu'elle renvoie. Cette phase a également inclus la conception de l'architecture globale de l'application et la définition des modèles de données nécessaires.

Phase 2 : Développement du backend (5 jours)

J'ai implémenté les fonctions Go pour communiquer avec l'API, gérer les erreurs et traiter les données. Cette phase incluait:

  • Création des structures de données pour représenter les cartes et les collections
  • Implémentation des fonctions pour récupérer et traiter les données de l'API
  • Mise en place du système de gestion des favoris avec persistance locale
  • Gestion des erreurs et mise en place de solutions de secours en cas d'indisponibilité de l'API

Phase 3 : Développement du frontend (4 jours)

J'ai créé les templates HTML et les styles CSS pour présenter les données de manière attrayante et responsive:

  • Conception de l'interface utilisateur avec un accent sur l'expérience utilisateur
  • Création des templates pour chaque page (accueil, liste des cartes, détails, etc.)
  • Développement du CSS pour un design moderne et adaptatif
  • Intégration de fonctionnalités JavaScript minimales pour améliorer l'interactivité

Phase 4 : Implémentation des fonctionnalités clés (4 jours)

J'ai implémenté les quatre fonctionnalités principales demandées:

  • Système de recherche par nom de carte
  • Système de filtrage par type, rareté et collection
  • Pagination des résultats avec options de nombre d'éléments par page
  • Gestion des favoris avec sauvegarde dans des fichiers JSON

Phase 5 : Tests et optimisations (3 jours)

J'ai testé l'application dans différents scénarios pour identifier et corriger les bugs:

  • Tests des différentes fonctionnalités et correction des bugs
  • Optimisation des performances, notamment pour le chargement des cartes
  • Amélioration de la gestion des erreurs et des cas limites
  • Ajout d'une page de diagnostic pour les problèmes d'images

Phase 6 : Documentation et finalisation (2 jours)

J'ai documenté le code et créé la documentation du projet:

  • Rédaction de commentaires explicatifs dans le code
  • Création du README avec instructions d'installation et d'utilisation
  • Développement de cette page "À propos" avec les détails du projet
  • Révision finale et préparation pour la soumission

Quels défis avez-vous rencontrés et comment les avez-vous surmontés ?

Ce projet m'a confronté à plusieurs défis techniques et organisationnels :

Limitation de l'API

Défi : L'API TCGdex ne propose pas nativement de pagination côté serveur ni de filtrage avancé.

Solution : J'ai implémenté ma propre logique de pagination et de filtrage côté client, en récupérant d'abord toutes les cartes puis en appliquant les filtres et la pagination manuellement. Pour optimiser les performances, j'ai mis en place un système de mise en cache des résultats.

Problèmes d'URLs d'images

Défi : Certaines URLs d'images renvoyées par l'API n'étaient pas directement utilisables ou complètes.

Solution : J'ai développé une fonction de correction des URLs qui ajoute les extensions manquantes et remplace les images non disponibles par des placeholders. J'ai également créé une page de diagnostic pour vérifier les problèmes d'images.

Gestion des erreurs API

Défi : L'API pouvait parfois être indisponible ou renvoyer des erreurs.

Solution : J'ai implémenté un système de retry avec backoff exponentiel, ainsi qu'une gestion des erreurs robuste avec des messages explicatifs pour l'utilisateur. J'ai également prévu des listes de secours pour les données essentielles comme les types de cartes.

Responsive design

Défi : Créer une interface qui s'adapte à tous les types d'appareils.

Solution : J'ai utilisé un système de grille flexible avec CSS Grid et des media queries pour adapter l'affichage selon la taille de l'écran. J'ai testé l'interface sur différents appareils pour assurer une expérience utilisateur optimale partout.

API : TCGdex

L'API TCGdex fournit des données complètes sur les cartes Pokémon, incluant les informations sur les cartes, les collections, les types et les raretés.

Endpoint Description Utilisation dans PokéTracker
/v2/en/cards Récupération de la liste des cartes Page des cartes, recherche et filtrage
/v2/en/cards/{id} Récupération des détails d'une carte Page de détail d'une carte
/v2/en/sets Récupération de la liste des collections Page des collections et options de filtrage
/v2/en/sets/{id} Récupération des détails d'une collection Page de détail d'une collection
/v2/en/types Récupération de la liste des types de cartes Options de filtrage par type
/v2/en/rarities Récupération de la liste des raretés de cartes Options de filtrage par rareté
